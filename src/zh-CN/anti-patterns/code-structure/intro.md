### 代码结构反模式

地道的 Rust 强调表达清晰的代码，充分利用语言特性。糟糕的代码结构会使 Rust 程序难以阅读、维护和调试，即使它们能成功编译。

本节介绍代码组织和风格中的常见反模式：

1- [手动索引](./manual-indexing.md)：使用基于索引的手动循环，而不是使用 `.iter()`、`.map()` 和 `.filter()` 等迭代器方法。

2- [糟糕的模式匹配](./poor-pattern-matching.md)：使用冗长的 `if-else` 链，而不是优雅的 `match` 表达式或 `if let`。

3- [不使用迭代器](./iterator-anti-patterns.md)：不必要地收集迭代器，或在迭代器链会更清晰、更高效时使用手动循环。

4- [忽视编译器警告](./ignoring-warnings.md)：忽略有用的编译器警告，这些警告指出潜在问题或非地道的代码。
