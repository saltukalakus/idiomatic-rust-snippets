### 忽视编译器警告

忽视或压制编译器警告（`#[allow(...)]` 或 `#![allow(...)]` 广泛使用）通常会掩盖潜在错误。警告是工具在编译期间提供的重要反馈，它们能提前发现逻辑错误、未使用变量、死代码或潜在的性能问题。

更好的做法：

- 修复根本原因，而不是抑制警告。
- 对于第三方生成代码或必须保持兼容性的特殊情况，尽量将 `allow` 的作用域限制到最小范围。

示例：

```rust, editable
// 不推荐：全局忽略导致真实问题被隐藏
#![allow(dead_code)]

fn unused_function() {
    // ...
}
```

```rust, editable
// 更好的做法：仅在确有需要时局部允许
#[allow(dead_code)]
fn helper_for_tests() { }
```
